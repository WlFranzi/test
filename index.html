Cornell Projects and Workshop Paper

academic

  • Cornell privateSpace:

    Technologies to automatically recognize stress and provide immediate support are extremely important to prevent chronic psychological stress. This project aims to do exactly that.

    We build an app which automatically detects when Cornell students are stressed through a wrist heartbeat sensor. Students are then encouraged to join a stress-managing social networking app that allows them to immediately engage in anonymous peer chat groups. Groups are formed by matching students dealing with similar issues. Project advised by Jean Costa.

    System Development
    Backend: NodeJS (GitHub)
    iOS Client: React Native (GitHub)

    Related Theory
    Appraisal theory, Theory of Diffusion of responsibility

    Franziska Wittleder, Christine Geeng, Xing Wei

  • CampusLife:

    This is a joined project across multiple institutes including Cornell University, Dartmouth College, Carnegie Mellon University and Georgia Tech. It focuses on using passive sensing data to automatically assess mental health states and academic performances of undergraduates.

    In this project, I lead the location analysis with the goal to understand daily mobility patterns of students and how anomalies in individual patterns correlate with the student's well-being. Project advised by Saeed Abdullah.

    Technologies Python, R

    Demo Data Visualization

  • Case Fiasco:

    When algorithm discriminate ... you need to trick them.

    My research paper will focus on location data and in particular how location data influences algorithms , hence, evokes information bias e.g. Google search results and Amazon price discrimination. I will also try to answer what a mobile users could do in order to receive less biased information. To provide a solution I will build a prototype of a faraday cage mobile phone case which will be directly attached to the phone. This case could help users to hide their location e.g. counseling meetings.
    Project advised by Professor Karen Levy.

    The final project will be published online Dec 15th.

  • CornellPulse:

    Everyday thousands of Cornell students use dining halls and fitness facilities on campus. To help Cornell increase its urban planning efficiency and give students the ability to better plan their daily schedules Cornell University has provided us with an API to compile, analyze and visualize the real time traffic data of Cornell's dining and fitness facilities.

    The app displays the current activity (idea: people/room capacity) for dining/fitness locations over time and compares it to the weekly average. I worked on bothfront-end and back-end to implement the algorithm for inferring activity scale.

    System Development
    Backend: NodeJS (AWS)
    iOS Client: React Native (GitHub)
    Website

    Jack Hayford, Franziska Wittleder, Matthew Crescimanno

    To support the COT I wrote a ruby/bash script. The script can be easily executed by users and checks Network, Transport and Application layer. The created log file will tell COT how to assist the user further.

  • SoundDrop:

    The open source project SoundDrop is a mobile web application that associates narrative audio clips with places in physical space - for example, to create a self guided walking tour of a city.

    SoundClouds HR Team aims to use this app as a tool to welcome SoundCloud members from abroad or potential new employees to Berlin and have them experience the city with a personalized SoundCloud tour.

  • Start-Up Weekend Zurich:
